Data Processing & Security Policy

Data Mirroring is build to prioritize the privacy and security of your data. This page outlines the security measures we have implemented to protect your information when using our application hosted on Heroku.

Coding for Security

Our application is built with a privacy-first approach. When you upload a file, it is temporarily stored in a secure location and only accessible during your session. Your data is automatically deleted within 30 minutes of inactivity, and all temporary files are completely removed when you close your browser or end your session. You can also manually remove all your data instantly by clicking the "End Session" button in the top navigation. We implement multiple cleanup mechanisms including per-request cleanup, periodic cleanup every 2 minutes, and server startup cleanup to ensure no persistent storage of personal data.

Our design is fully open-source, please see: Data Mirroring Github Page.

Hosting & Infrastructure Security

Our application is hosted on Heroku in EU-based data centers. Heroku manages infrastructure, scaling, and platform security, applying industry best practices and rapid security updates. All data is encrypted in transit and at rest, and Heroku maintains compliance certifications such as ISO 27001, SOC 2, and PCI DSS.

For more details, please visit: Heroku's Security Policy.

When accessing our tool from data-mirror.org, all traffic is routed through Cloudflare, which provides:

  • TLS/SSL encryption for all connections
  • Unmetered DDoS protection and IP-based rate limiting
  • Web Application Firewall (WAF) with protections against common high-severity vulnerabilities
  • Bot protection to block automated threats
  • Global CDN to ensure performance and availability worldwide

Additionally, our application enforces strict file type restrictions for uploads to minimize risks from malicious content.

For more details please visit Secure & accelerate applications with Cloudflare.

GDPR Compliance & Data Privacy

We rely on highly trusted and secure third-parties to be fully GDPR compliant and ensure:

  • All data is processed in the EU, preventing international data transfers.
  • No persistent storage of personal data.

For more details, refer to: Heroku GDPR Compliance and Cloudflare and GDPR compliance.

Key Takeaways

  • Our application is designed with a privacy-first approach.
  • We use Heroku EU-based servers for compliance with GDPR.
  • Expedited WAF protects against cyber threats.
